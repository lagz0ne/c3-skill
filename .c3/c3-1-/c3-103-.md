# schema

## Goal

Define the canvas types a fact may be — built-in and project-declared — and validate that a fact's body matches its canvas shape.

## Parent Fit

| Field | Value |
| --- | --- |
| Parent | c3-1 |
| Role | The shape authority: it owns what sections, column types, and edges a given entity type requires, and resolves the canvas any fact is checked against. |
| Boundary | Owns canvas definitions (embedded seeds plus project-local .c3/canvases/*.md) and canvas validation; it does not parse fact bodies into nodes (doc-model) or persist anything (store). |
| Collaboration | doc-model reads the edge-columns this layer exposes to sync relationships; check resolves a fact's canvas here and validates required sections and typed columns against it; the change-doc lifecycle keys on the status legal-set this layer reads from a canvas. |

## Purpose

Hold every canvas definition: the embedded built-in canvases compiled in via go:embed, plus user-owned project canvases loaded from `.c3/canvases/`, with the project file overriding the built-in for the same type. Parse and validate a canvas document — unique named sections, known content types, supported column primitives, and at most one edge-writer column per relationship type — and recognize a change doc by its declared status legal-set. Non-goals: parsing or rendering fact markdown (doc-model), and storing or querying entities (store).

## Governance

| Reference | Type | Governs | Precedence | Notes |
| --- | --- | --- | --- | --- |
| ref-frontmatter-docs | ref | The frontmatter-plus-markdown shape a canvas document itself uses (id, type: canvas, description, status) | Convention defines the canvas file format | A canvas's frontmatter is restricted to id, type, description, status, and c3-seal; everything else is rejected. |
