# codemap-lib

## Goal

Match files against source-path globs and detect glob patterns — the resolver eval and lookup use to map a fact's declared code surface to real files.

## Parent Fit

| Field | Value |
| --- | --- |
| Parent | c3-1 |
| Role | The code-binding layer: it owns reading the code-map file and the glob matching that ties a source file to the component, ref, or rule that governs it. |
| Boundary | Owns code-map parsing, glob expansion, and binding validation; it decides nothing about doc shape (schema) and persists nothing (store) — it works against the filesystem and a passed entity-type map. |
| Collaboration | check passes it the entity-id-to-type map from the graph and the project dir, then surfaces the issues it reports; lookup uses its glob matching to map a file back to its governing entity. |

## Purpose

Expand a glob against the project filesystem with doublestar, including a literal-bracket fallback so `[id]`-style framework paths still match, and report whether a path is a glob pattern. Eval calls this to resolve each fact's `code:` binding into the files to check; lookup calls it to map a file back to its owning fact. Non-goals: parsing fact bodies (doc-model), deciding canvas shape (schema), or reading the database (store).
