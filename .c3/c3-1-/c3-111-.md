# author-cmds

## Goal

Create and edit facts and canvases before they freeze: scaffold a new project, add an entity from a body, replace content or fields, and author canvas definitions.

## Parent Fit

| Field | Value |
| --- | --- |
| Parent | c3-1 |
| Role | The pre-freeze authoring surface: the commands that bring a fact into existence and shape it while it is still directly editable. |
| Boundary | Owns validating a body against its canvas and writing it through the store; it stops at the freeze line — once a fact carries a sealed body, these commands defer to the change-unit. |
| Collaboration | schema supplies the canvas each body is validated against; doc-model turns markdown into the node tree the store persists; the store assigns ids, writes nodes, and reconciles membership. |

## Purpose

Serve the create/edit side of C3: `init` scaffolds `.c3/` with the seed canvases and the genesis system + adoption ADR, `add` builds a typed entity from a stdin body (validating before any write and reconciling the parent's membership table), `write`/`set` replace a body, a section, or a frontmatter field, and `schema`/`canvas` show and author canvas definitions. Non-goals: mutating a frozen fact (change-cmds owns that), applying a change-unit, or querying the graph (read-cmds).

## Governance

| Reference | Type | Governs | Precedence | Notes |
| --- | --- | --- | --- | --- |
| rule-output-via-helpers | rule | A command's structured result serializes through the shared helpers, not a hand-rolled print | Format selection is centralized | add/canvas/codemap route results through WriteObjectOutput / WriteTableOutput. |
| rule-wrap-error-cause | rule | A write or validation failure returned to the caller wraps its underlying cause with operation context | The failure chain stays unwrapped-reachable | add/write wrap store and content errors with fmt.Errorf(... : %w, err). |

## Contract

| Surface | Direction | Contract | Boundary | Evidence |
| --- | --- | --- | --- | --- |
| add / write / set | IN | Callers pass a type+slug+body or an id+content; the body is validated against its canvas before any DB write, and a failing validation aborts with no partial state | A create that fails after insert compensates by deleting the entity — never a half-written fact | cli/cmd/add.go RunAddFormattedInDir; cli/cmd/write.go RunWrite |
| init / canvas / codemap | OUT | Scaffolds the store and canvases, persists an authored canvas as sealed markdown, or records code-map globs, reporting what was created or changed | Write-if-absent on seed canvases; status stays edit-proof through a body write | cli/cmd/init.go RunInitDB; cli/cmd/canvas.go RunCanvas |
