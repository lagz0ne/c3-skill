# @c3x/cli Runtime Manager Plan

**Goal:** make the npm `c3x` entrypoint a runtime manager for GitHub Release assets while keeping
normal C3 commands as transparent passthrough to the Go binary.

**Distribution boundary:** the npm client remains thin and manages runtime assets. The release also
ships a platform-neutral no-binary skill/plugin ZIP that delegates through the pinned npm manager,
per-platform full-fat skill ZIPs with embedded semantic runtime for sandboxed and offline use, and
Linux portable fat ZIPs with a bundled pure-Go core binary for broader distro/sandbox compatibility.

**Spec:** `docs/superpowers/specs/2026-03-17-c3x-npm-cli-design.md`

## Scope

- Add `c3x runtime ...` commands for version listing, install, project selection, uninstall, and
  explicit cache pruning.
- Resolve normal command execution from project operational metadata or the latest GitHub Release.
- Keep project metadata limited to `.c3/runtime.json` with a version only.
- Keep runtime asset trust anchored in HTTPS release assets plus SHA256 verification.
- Preserve normal passthrough behavior for commands such as `c3x list`, `c3x check`, and
  `c3x search`.
